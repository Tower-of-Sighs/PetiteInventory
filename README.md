## 娇小物品栏

提供了类似生化危机4、暗黑破坏神、三角洲行动的容器布局，即一项物品占用多个槽位，并改动了相应的替换和转移操作。

### 主要功能

首次启动后会生成默认规则：config/PetiteInventory/default.json

规则的写法非常简单易懂，看一眼就会了，支持标签匹配，/reload指令重载。

进入游戏后即可看到物品栏布局发生了变化：

![img](https://resource-api.xyeidc.com/client/pics/30b839f7)

出于游戏平衡考虑，新布局默认只对玩家物品栏生效，如果不希望对玩家物品栏生效，可以在配置文件中关闭。

如果需要对其它容器启用新布局，对着目标容器的任意槽位按下复制热键（默认为U），即可复制目标容器的容器标识。

将容器标识粘贴到配置文件中，即可对其启用本模组提供的新布局。

此外，对于已开启新布局的容器，还有以下操作上的优化：

- 鼠标移动一种物品替换另一种物品时，只要目标区域内仅有一种其它物品，即可直接替换。
- SHIFT+鼠标右键快速转移物品时，若无法堆叠，则会智能寻找可用空白区域放置物品。
- 无论如何，快捷栏不会受到任何影响，不然就太折磨了。

### 注意事项

- 本模组的性质决定了注定会多多少少和其他模组产生兼容性问题，如容器整理、槽位拓展等。
- 本模组与容器整理注定不会有理想的兼容方案，矛盾不仅在于程序兼容，还在于本模组设计初衷之一的手动整理。
- 在启用了新布局的容器中，按住鼠标拖拽物品的行为被禁用，如果未来攻克了难题再考虑回归。
- 对于其他模组的重要容器，如果存在功能失效的现象，请留言。
